RLS POLICIES:
- tournaments: spectators SELECT where status IN ('published','in_progress','completed')
- matches: spectators SELECT where tournament status is public
- team_out_dates: team managers SELECT/INSERT/UPDATE/DELETE where their assignment exists AND tournament status != 'published'. After published: SELECT only.

- All write operations: require authenticated user with appropriate role

========================================================
SCHEDULING ALGORITHM — /lib/scheduler.ts
========================================================

Implement as a pure TypeScript function:
generateSchedule(tournament, teams, playDates, fields, outDates): { matches: Match[], warnings: Conflict[] }

ROUND ROBIN: Berger/Circle method for all pairings. Greedy slot-fill across (date, timeslot, field).
Constraints:
- No team plays twice in the same timeslot
- Respect team_out_dates for every team (hard constraint)
- Enforce max_games_per_day per team
- Balance field usage when configured
- Locked matches (is_locked=true) must not be moved on re-generate

SINGLE ELIMINATION: Generate bracket with byes for non-power-of-2. Place earlier rounds on early dates, finals on tournaments.finals_date. Seed by handicap or manual seed value.

DOUBLE ELIMINATION: Two-loss elimination. Winners bracket + losers bracket. Total matches = 2N-1 or 2N-2.

GROUP + KNOCKOUT: Run round robin scheduler for each group independently. Reserve knockout slots on later play dates.

SWISS: Generate Round 1 randomly. Future rounds are placeholders (home_team_id/away_team_id null) until prior round scores entered. Fill subsequent rounds by standings after each round.

CONFLICT HANDLING: Never throw. Return warnings array.
- type: 'error' = hard conflict (same team double-booked, team has all dates blocked). Blocks publishing.
- type: 'warning' = soft conflict (fewer games than expected due to constraints). Admin can proceed.
- If a team has ALL available play dates in team_out_dates: return error naming that team specifically.

      "match_counts": { "group": 6, "knockout": 3, "final": 1 },

    }

}

feasibility values: "ideal" | "viable" | "tight" | "not_feasible"

Parse response strictly. Strip any markdown fences before parsing.

Display in Step 6 format picker:

- feasibility "ideal": no badge
- feasibility "tight": amber "Tight" label

CLIENT-SIDE CLOCK FORMULA (use in spectator app too):
const computeClockSeconds = (match) => {
  if (!match.clock_is_running) return match.clock_elapsed_seconds;
  return match.clock_elapsed_seconds +
    Math.floor((Date.now() - new Date(match.clock_started_at).getTime()) / 1000);
}

  GET /api/export/schedule/[tournamentId] — returns PNG
  GET /api/export/daily/[tournamentId]?date=YYYY-MM-DD — returns PNG
  GET /api/export/pdf/[tournamentId] — returns PDF

Each route: launch Puppeteer, navigate to matching /render/* URL, screenshot at deviceScaleFactor:2, pipe PNG buffer as response with Content-Type: image/png.

Export dimensions:
  Bracket: 1920x1080 (landscape)
  Full schedule: 2400x1600 (landscape)
  Daily fixture sheet: 1080x1350 (portrait, social-friendly)
  Standings: 1080x1350 (portrait)

Cache exports in Supabase Storage for 1 hour at /exports/[tournamentId]/[type]/[hash].png

========================================================
SUPABASE EDGE FUNCTIONS
========================================================

1. on_match_live — trigger when matches.status changes to 'live'
   Read spectator_follows WHERE club_id matches tournament's club.
   Read push_subscriptions for those user_ids.
   Send FCM batch notification: "[Team A] vs [Team B] is now live on Field [N]"

2. on_match_final — trigger when matches.status changes to 'final'
   If knockout match: read bracket_position, find next match slot, update that match's
   home_team_id or away_team_id with the winner's team_id.
   Send FCM: "Final score: [Team A] [score] - [score] [Team B]"

3. on_club_logo_upload — trigger on Supabase Storage upload to /clubs/ path
   Sharp.js resize to 400x400, 96x96, 40x40.
   Update clubs table with all three URLs.

4. on_team_logo_upload — trigger on Supabase Storage upload to /teams/ path
   Sharp.js resize to 400x400, 64x64.
   Update teams table with logo_url and logo_thumb_url.

5. send_invite_email — called when creating user_invites row
   Use Resend to send invite email.
   For team manager: subject "[Club Name] has added you as Team Manager for [Team Name]"
   For club manager: subject "You have been appointed Club Manager for [Club Name]"
   Include magic link to /accept-invite?token=[token]
