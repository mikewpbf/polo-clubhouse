DATA REQUIREMENTS
The page receives :matchId and :playerId. It must fetch in a single round 
trip (broadcast graphics should make minimum requests):

Add a new endpoint:
  GET /matches/:matchId/player-stats/:playerId
returning:
  {
    tournamentName: string,
    player: { id, name, headshotUrl, teamId, teamName, teamLogoUrl },
    match: {
      goals, shotsOnGoal, penaltyGoals, throwInsWon
    },
    tournament: {
      goals, avgPerMatch (number, 1 decimal), shotsOnGoal,
      conversion (number, integer percent — goals / shotsOnGoal × 100, 
        rounded to integer; 0 when shotsOnGoal is 0)
    }
  }

Computation rules:
- All match stats: count match_events for this match where player_id 
  matches and event_type matches the stat (e.g., 'goal', 'shot_on_goal', 
  'penalty_goal', 'throw_in_won').
- All tournament stats: same queries scoped to tournament.matches instead 
  of one match.
- Avg per Match: tournament goals / count of distinct matches in this 
  tournament where this player appears in team_players for the season. 
  Round to 1 decimal. If denominator is 0, return 0.0.
- Conversion: tournament goals / tournament shotsOnGoal × 100, rounded 
  to integer percent. If shotsOnGoal is 0, return 0.

Note: shotsOnGoal must already be tracked at the player level via 
match_events. If it isn't, this is a separate prerequisite — the prompt 
should flag and stop rather than make up data. Verify before building.
